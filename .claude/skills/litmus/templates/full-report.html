## Substitution rules Hyphens in path segments (e.g. `categories.content-discoverability`) are literal key characters, not subtraction operators. All field references that exist in `templates/full-report.md` resolve the same way here. - `{{ts}}`, `{{hostname}}`, `{{input_url}}`, `{{skill_version}}`, `{{conversion_method}}` — from `manifest.json`. - `{{readability}}`, `{{readability_partial}}`, `{{readability_unavailable}}` — truthy when the corresponding manifest block is populated. Sub-keys map directly (e.g. `{{readability.overall_score}}`, `{{readability_partial.pages_tested}}`). - `{{readability.overall_grade_lower}}` — `manifest.readability.overall_grade` lowercased (used as a CSS class name: `a`, `b`, `c`, `d`, `f`). - `{{readability.diagnostics}}` — array from `manifest.readability.diagnostics` (AFDocs `.scoring.diagnostics`). Each entry has `id`, `severity` (`warning` or `critical`), `message`, `resolution`. Block is omitted when empty. - `{{readability.failed_checks}}` — array from `manifest.readability.failed_checks`. Each entry has `id`, `category`, `status` (one of `fail`, `warn`), `message`, and optional `resolution` (string or null) joined from AFDocs' `.scoring.resolutions[id]`. Block is omitted when empty. - `{{score}}`, `{{grade}}`, `{{grade_lower}}`, `{{passed}}`, `{{failed}}`, `{{errored}}`, `{{total}}` — computed from `evaluations.json`. `{{grade_lower}}` is `{{grade}}` lowercased. - `{{overall_grade}}` — combined per the rule in `SKILL.md`. `{{overall_grade_lower}}` is its lowercased form. - `{{overall_marker}}` — the parenthetical suffix when one axis is partial or unavailable (e.g. `(readability partial)`, `(readability unavailable)`). Empty when both axes are present. - `{{prioritized_sections}}` — group `evaluations[]` where `status === "failed"` by `responsible_section`, sort by group size desc, then by section slug asc. Each entry: `rank`, `section_slug`, `failure_count`, `failures_in_section[]`. - `{{no_failures}}` — boolean: `failed === 0` AND `errored === 0`. - `{{tasks_with_evaluations}}` — `tasks.json` joined with `evaluations.json` on `task_id`. Order: by `id` asc. Each entry exposes `id`, `description`, `success_criterion`, `status`, `exit_code`, `duration_ms`, `stdout_excerpt`, and (when failed) `root_cause`, `responsible_section`, `fix_suggestion`, plus (when errored) `error_phase`, `error_message`. - `{{task_count}}` — `tasks.json.length`. - `{{stdout_excerpt}}` — first 500 chars of `evidence.stdout`; truncate with `…` if longer. ## Edge cases - **No execution failures.** Render the `{{#if no_failures}}` block instead of the prioritized fix list. Both blocks must not appear together. - **Both `readability` and `readability_partial`/`readability_unavailable` populated.** Per `SKILL.md` Decision Gates the run halts before rendering. If reached, render the populated state and skip the others. - **HTML escaping.** Treat all `{{...}}` substitutions as plain text by default. `fix_suggestion`, `description`, and `stdout_excerpt` may contain `<`, `>`, `&`, or backticks — escape to `<`, `>`, `&`, `` ` `` when substituting into HTML body content. Inside `` and `
` blocks, escape `<` and `>` but leave entities readable.
- **Brand mark.** The inline SVG is part of the template and is rendered verbatim, no substitution inside it.
- **External fonts.** The template references Inter and JetBrains Mono by name but does NOT `@import` them. Browsers fall back to system fonts.

## Length budget

Target ≤ 8000 lines for a 10-task run. If the rendered report exceeds this, the renderer truncates per-task `stdout_excerpt` further (down to 200 chars) and flags the truncation in the task-body.
