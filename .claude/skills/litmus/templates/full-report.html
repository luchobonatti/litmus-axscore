## Substitution rules Hyphens in path segments (e.g. `categories.content-discoverability`) are literal key characters, not subtraction operators. All field references that exist in `templates/full-report.md` resolve the same way here. - `{{ts}}`, `{{hostname}}`, `{{input_url}}`, `{{skill_version}}`, `{{conversion_method}}` — from `manifest.json`. - `{{readability}}`, `{{readability_partial}}`, `{{readability_unavailable}}` — truthy when the corresponding manifest block is populated. Sub-keys map directly (e.g. `{{readability.overall_score}}`, `{{readability_partial.pages_tested}}`). - `{{readability.overall_grade_lower}}` — `manifest.readability.overall_grade` lowercased (used as a CSS class name: `a`, `b`, `c`, `d`, `f`). - `{{readability.diagnostics}}` — array from `manifest.readability.diagnostics` (AFDocs `.scoring.diagnostics`). Each entry has `id`, `severity` (`warning` or `critical`), `message`, `resolution`. - `{{readability.failed_checks}}` — array from `manifest.readability.failed_checks`. Each entry has `id`, `category`, `status` (`fail` or `warn`), `message`, and optional `resolution` (string or null) joined from AFDocs' `.scoring.resolutions[id]`. - `{{readability_categories}}` — derived array of 7 entries, one per AFDocs category. Each: `key` (e.g. `content-discoverability`), `label` (human-readable, e.g. `Content discoverability`), `score` (0-100 from `readability.categories[key]`), `grade_band` (one of `a`/`b`/`c`/`d`/`f` from score-to-grade mapping). - `{{score}}`, `{{grade}}`, `{{grade_lower}}`, `{{passed}}`, `{{failed}}`, `{{errored}}`, `{{total}}` — computed from `evaluations.json`. - `{{overall_grade}}` — combined per the rule in `SKILL.md`. `{{overall_grade_lower}}` is the lowercased CSS-class form. - `{{overall_marker}}` — parenthetical suffix when one axis is partial or unavailable. Empty when both axes are present. - `{{axes_summary}}` — short summary string of the two axes. Format: `Readability · Execution ` when both present; `Execution (readability partial/unavailable)` when one axis is missing. - `{{has_findings}}` — boolean: at least one of `readability.diagnostics`, `readability.failed_checks`, or `execution_findings` is non-empty. - `{{no_findings}}` — boolean: all three of the above are empty. - `{{execution_findings}}` — flat array derived from failed `evaluations[]`. Each entry: `task_id`, `description` (from `tasks.json`), `root_cause`, `responsible_section`, `fix_suggestion`. Order: by `task_id` asc. - `{{tasks_with_evaluations}}` — `tasks.json` joined with `evaluations.json` on `task_id`. Order: by `id` asc. Each entry exposes `id`, `description`, `success_criterion`, `status`, `exit_code`, `duration_ms`, `stdout_excerpt`, and (when failed) `root_cause`, `responsible_section`, `fix_suggestion`, plus (when errored) `error_phase`, `error_message`. - `{{task_count}}` — `tasks.json.length`. - `{{stdout_excerpt}}` — first 500 chars of `evidence.stdout`; truncate with `…` if longer. ## Edge cases - **No findings at all.** Render the `{{#if no_findings}}` block instead of the per-sub-block iteration. - **Both `readability` and `readability_partial`/`readability_unavailable` populated.** Per `SKILL.md` Decision Gates the run halts before rendering. If reached, render the populated state and skip the others. - **HTML escaping.** Treat all `{{...}}` substitutions as plain text by default. `fix_suggestion`, `message`, `resolution`, `description`, and `stdout_excerpt` may contain `<`, `>`, `&`, or backticks — escape to `<`, `>`, `&`, `` ` `` when substituting into HTML body content. - **Brand mark.** The inline SVG is part of the template and is rendered verbatim, no substitution inside it. - **External fonts.** The template references Inter and JetBrains Mono by name but does NOT `@import` them. Browsers fall back to system fonts. ## Length budget Target ≤ 10000 lines for a 10-task run. If the rendered report exceeds this, the renderer truncates per-task `stdout_excerpt` further (down to 200 chars) and flags the truncation in the task-body.
