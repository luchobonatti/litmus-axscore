## Substitution rules Hyphens in path segments (e.g. `categories.content-discoverability`) are literal key characters, not subtraction operators. All field references that exist in `templates/full-report.md` resolve the same way here. - `{{ts}}`, `{{hostname}}`, `{{input_url}}`, `{{skill_version}}`, `{{conversion_method}}` — from `manifest.json`. - `{{date_human}}` — long human-readable date and time derived from `manifest.ts` (e.g. `May 20, 2026 · 13:33 UTC`). - `{{date_compact}}` — short ISO date derived from `manifest.ts` (e.g. `2026-05-20 · 13:33 UTC`). - `{{readability}}`, `{{readability_partial}}`, `{{readability_unavailable}}` — truthy when the corresponding manifest block is populated. - `{{readability.overall_grade_lower}}` — `manifest.readability.overall_grade` lowercased (used as CSS class suffix `g-a`...`g-f`). - `{{readability_summary}}` — short string for the metastrip: ` /100` when readability is present, `partial` / `unavailable` otherwise. - `{{readability.diagnostics}}`, `{{readability.failed_checks}}` — arrays as documented in `SKILL.md`. Each entry passed to `{{#each}}` is augmented with `exhibit_num` (zero-padded sequential number across the entire Exhibits section, e.g. `01`, `02`, …). - `{{readability_categories}}` — derived array, 7 entries, each with `key`, `label` (humanized), `score`, `grade_band` (lowercased `a`...`f`), `grade_letter` (uppercased). - `{{score}}`, `{{grade}}`, `{{grade_lower}}`, `{{passed}}`, `{{failed}}`, `{{errored}}`, `{{total}}` — computed from `evaluations.json`. - `{{overall_grade}}`, `{{overall_grade_lower}}`, `{{overall_marker}}` — combined per `SKILL.md` rules. - `{{axes_summary}}` — editorial sentence summarizing the verdict, may include `...` emphasis spans around key noun phrases. - `{{has_findings}}` / `{{no_findings}}` / `{{findings_count}}` / `{{findings_count_caps_s}}` — drive the Exhibits section. `findings_count_caps_s` is `S` (uppercase) when plural, empty string when 1, matching the eyebrow's all-caps styling. - `{{execution_findings}}` — flat array derived from failed `evaluations[]` joined with task description, augmented with `exhibit_num`. - `{{tasks_with_evaluations}}` — `tasks.json` joined with `evaluations.json`. Order: by `id` asc. - `{{task_count}}` — `tasks.json.length`. - `{{stdout_excerpt}}` — first 500 chars of stdout, truncated with `…` if longer. ## Edge cases - **No findings at all.** Render the `{{#if no_findings}}` block instead of the per-sub-block iteration. - **Both `readability` and `readability_partial`/`readability_unavailable` populated.** Per `SKILL.md` Decision Gates the run halts before rendering. If reached, render the populated state and skip the others. - **HTML escaping.** Treat all `{{...}}` substitutions as plain text. `fix_suggestion`, `message`, `resolution`, `description`, and `stdout_excerpt` may contain `<`, `>`, `&` — escape to `<`, `>`, `&` when substituting into HTML body content. - **Brand mark.** The five `` elements inside `.lockup-bar` render the segmented gradient strip; they are part of the template and rendered verbatim with no substitution inside. - **External fonts.** The template loads Inter and JetBrains Mono from Google Fonts via ``. Falls back to system fonts when offline. For fully offline-safe rendering, a future revision may inline the fonts as base64. ## Length budget Target ≤ 10000 lines for a 10-task run. If the rendered report exceeds this, the renderer truncates per-task `stdout_excerpt` further (down to 200 chars) and flags the truncation in the task-body.
